# syncstore

Go-библиотека: значение в памяти, обновляемое по сигналу из внешнего
источника. Типовой сценарий — реплики сервиса держат небольшую таблицу
(справочник, конфигурацию) в памяти и перечитывают её, когда источник
сигналит «данные изменились».

Ядро (`github.com/HiveTraum/syncstore`) агностично к хранилищу и без
внешних зависимостей. Контракт — три роли:

- **`Listener`** — источник сигналов «данные изменились» (интерфейс);
- **`Loader[T]`** — функция, читающая актуальное значение целиком;
- **`Store[T]`** — хранилище: отдаёт значение из памяти, по сигналу
  перечитывает его Loader'ом.

Конкретные СУБД и брокеры подключаются **драйверами** — отдельными
Go-модулями в подпакетах репозитория (со своим `go.mod`), реализующими
`Listener`. Ядро не тянет их зависимости.

## Использование

```go
import "github.com/HiveTraum/syncstore"

type Rate struct{ Currency string; Value float64 }

store := syncstore.New(
    listener, // слушатель сигнала — драйвер источника (syncstore.Listener)
    func(ctx context.Context) ([]Rate, error) { // чтение данных
        return queries.AllRates(ctx)
    },
)

go store.Run(ctx) // одна горутина на процесс: слушает и перечитывает

rates, err := store.Get(ctx) // значение из памяти, целиком
```

`Get` отдаёт закэшированное значение; если хранилище ещё пустое (например,
`Run` не успел загрузиться) — синхронно загружает его тем же Loader'ом.
Значение разделяется всеми вызывающими: если `T` содержит slice/map,
обращайтесь с ним как с read-only.

Сигнал — **чистый триггер перечитки**, данных не несёт. Поэтому потеря или
схлопывание сигналов безопасны: любая перечитка приводит хранилище к
актуальному состоянию.

### Опции

- `WithDebounce(d)` — схлопывать пачку сигналов в одну перечитку;
- `WithRetryInterval(d)` — пауза перед повтором неудачной перезагрузки
  (по умолчанию 5s; старое значение при ошибке сохраняется);
- `WithOnError(fn)` — логирование/метрики ошибок фоновых перезагрузок.

## Контракт драйвера (`Listener`)

- `Listen(ctx)` запускает прослушивание и возвращает **канал сигналов**;
  канал закрывается, когда прослушивание останавливается (отмена `ctx` или
  невосстановимая ошибка — тогда `Run` вернёт `ErrListenerStopped`).
  Переподключения после обрывов — забота драйвера.
- Если за время обрыва сигналы могли быть потеряны, драйвер **обязан выдать
  синтетический сигнал** после восстановления — хранилище перечитает данные
  и догонит пропущенное. Благодаря семантике «сигнал = триггер» это
  единственное, что требуется для корректности.

## Ограничения

- Синхронизация — **eventual**: между изменением данных и перечиткой реплики
  отдают старое значение. Для справочников/конфигурации это норма.
- Значение перечитывается **целиком** — механизм рассчитан на небольшие
  объёмы. Точечная перечитка изменившейся части — осознанно за рамками
  (см. roadmap продукта).
